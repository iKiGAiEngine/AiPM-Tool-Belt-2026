2026 AiPM v3 — Full Platform
8+ Modules Live · In Active Use

Built around one core principle: if it already exists digitally, use it — never type anything twice. Proposal Log data carries directly into the estimating module with little to no setup. By centralizing everything in structured digital form, the platform now creates the foundation for expanded data analytics, deeper automation, and future AI integration opportunities that weren't possible when the work lived in spreadsheets.

04 · Front End +

AI Document Parsing

Spec books and schedules, parsed instantly.

Two parsing engines built into the same workflow. The Spec Extractor scans a full project spec book, identifies every Division 10 section, and exports each one as a labeled PDF organized by scope, CSI number, and project. The Schedule Extractor takes any schedule — screenshot, pasted text, or PDF — and outputs structured line items ready to use. Both run inside the estimating module: spec sections get filed automatically, schedule data populates material line items in their corresponding scope sections. Work that previously required a project engineer to prep before estimating could begin — now done by the estimator directly, in minutes. Used multiple times daily.

02 · Back End +

Layered Backup & Recovery

Four overlapping safety nets.

Nightly database backups run automatically. GitHub commits capture every deploy. Manual pre-deploy snapshots are taken before any major change. A Google Sheet mirror of live data — covering vendors, manufacturers, the proposal log, and user profiles — runs nightly as a final fallback. The whole stack gets audited on a routine schedule to confirm the safety net is intact.

Nightly DBGitHubPre-Deploy SnapshotMirror Sheet
03 · Back End +

Full Rollback Capability

Every change reversible, root cause traceable.

If a deploy goes sideways, the platform can be restored to any prior state without losing live data. The audit log captures who changed what — so root cause is always traceable. Versioning runs separately from GitHub for the database itself, with manual checkpoints before any larger update where the outcome isn't fully predictable.

GitHubAudit LogPoint-in-Time Recovery
